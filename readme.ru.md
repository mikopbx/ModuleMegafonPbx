## ModuleMegafonPbx — загрузка истории звонков из MegaPBX в MikoPBX

Коротко:

- **Назначение**: импорт истории звонков из MegaPBX в CDR MikoPBX.
- **Дальше**: из MikoPBX историю можно выгружать в 1С:Предприятие (см. `getHistory.epf`).
- **Ограничение**: интеграция работает только с MegaPBX; модуль читает историю и не управляет звонками.

### Требования
- Доступ к API MegaPBX (CRM API) и актуальный токен/учетные данные.
- Установленный MikoPBX с правами установки и запуска модулей.

### Установка
1. В веб‑интерфейсе MikoPBX перейдите: Модули → Маркетплейс модулей → Загрузить новый модуль.
2. Выберите архив модуля `ModuleMegafonPbx` и дождитесь установки.
3. Активируйте модуль.

### Настройка
В настройках модуля укажите:
- **Адрес АТС Мегафон**: например `vats.megafon.ru`.
- **Ключ для авторизации API** MegaPBX.
- **Какой номер отображать в истории** - внутренний или мобильный номер сотрудника.
- **Сдвиг времени** - позволяет скорректировать время на указанное количество часов.

Сохраните изменения.

Скрипт использует сохраненные в модуле настройки подключения и период синхронизации.

### Как это работает
- Модуль обращается к API истории MegaPBX и получает события за период.
- Записи добавляются в CDR MikoPBX; предусмотрены механизмы для предотвращения дублей.
- Модуль работает только на чтение и не изменяет данные в MegaPBX.

### Экспорт в 1С:Предприятие
- Историю звонков, импортированную в MikoPBX, можно выгружать в 1С стандартными средствами.
- В репозитории присутствует обработка `getHistory.epf` для загрузки истории из MikoPBX в 1С.
